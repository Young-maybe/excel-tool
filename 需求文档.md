### 项目需求文档（总览）

本文档用于**快速理解项目脉络**、统一需求边界、方便未来新会话/新维护者在最短时间内接手。

---

### 1. 项目定位与目标

- **产品形态**：Windows 10/11 本地桌面应用（Python + PyQt6），单窗口。
- **目标用户**：非技术用户，通过图形界面处理 Excel/CSV。
- **核心目标**：
  - 左侧模块导航、右侧模块操作区
  - 用户手动选择输入（文件夹或单个文件）
  - 输出统一写入用户配置的“输出目录”
  - 稳定、可维护、可回退、可打包为 exe

---

### 2. 关键开发约束（必须遵守）

来源：`.cursor/rules/forapp.mdc`

- **UI 与业务逻辑严格分离**
  - `ui/`：只放界面逻辑（选择输入、触发执行、展示提示）
  - `modules/`：只放业务处理逻辑（输入→输出），不弹窗、不依赖 UI
- **不动态加载外部源脚本**
  - 所有“你提供的源文件夹”都视为将被删除
  - 必须把核心逻辑迁入 `modules/` 内部
- **输入/输出统一策略**
  - 输入从“选定文件夹/文件”获取，不再读取“当前目录”
  - 输出统一写入“输出目录”的模块子文件夹
- **避免 WinError 32**
  - 复制到模块工作目录再处理，避免占用与同名冲突
- **不写测试文件**
  - 除非你明确要求做测试

---

### 3. 工程目录结构（核心文件）

- `main.py`
  - 程序入口、日志与全局异常处理
- `ui/main_window.py`
  - 主窗口：左侧 `QTreeWidget` 导航 + 右侧内容区（`QStackedWidget` + `QScrollArea`）
  - 左下角“设置输出文件夹”
- `ui/components/`
  - `folder_task_page.py`：**选择文件夹 → 运行任务**通用页
  - `file_task_page.py`：**选择单文件 → 运行任务**通用页
  - `apple_progress_panel.py`：Apple 风格进度 + 状态日志面板
  - `task_runner.py`：QThread 后台执行框架（避免 UI 卡死，支持 `progress_cb`）
  - `dialogs.py`：统一“选择文件夹”弹窗（Qt 非原生 + 马卡龙主题 + 默认桌面/侧边栏快捷入口）
  - `manual_text_repo.py`：从 `huancuntxt/` 读取可维护的使用手册文本
  - `usage_stats_page.py`：使用统计二级页面（树形报表 + 返回）
  - `stock_tool_legacy_main.py` / `stock_tool_embedded_page.py`：库存处理工具的内嵌版（保留原 UI/逻辑）
- `modules/`
  - 各业务模块逻辑（见“模块清单”）
- `services/`
  - `usage_stats.py`：使用统计（节省时长）持久化与报表聚合
- `huancuntxt/`
  - 可维护的使用手册文本（每个模块一个 `.txt`，可直接编辑）
- `问题清单.md`
  - 已踩坑/解决方案/经验记录（后续维护时优先查这里）

---

### 4. 主界面交互与统一“输出目录”

- **设置输出目录**：主界面左下角按钮“⚙ 设置输出文件夹”
- **各模块输出路径**：
  - 全部写入 `输出目录/<模块名或模块子目录>/...`
  - 若存在同名或文件占用：自动改名为 `*_处理副本N.xlsx`
- **文件夹选择弹窗（马卡龙适配）**：
  - 输入/输出的“选择文件夹”统一使用 Qt 非原生对话框（可被 QSS 控制）
  - 默认打开桌面，并在左侧快捷栏加入“桌面/文档/下载/用户目录”

---

### 5. 统一的“进度百分比 + 处理状态提示”（除库存处理工具外）

目标：所有耗时模块在右侧页面显示：
- **百分比进度条**（0~100）
- **阶段状态文本**（例如：扫描输入/复制文件/生成输出）
- **详细日志滚动**
- **UI 不冻结**：任务在后台线程执行

实现：
- UI 页面：`ui/components/folder_task_page.py`、`ui/components/file_task_page.py`
  - 内嵌 `AppleProgressPanel`
  - 点击开始后通过 `task_runner.py` 的 `QThread` 执行 handler
- 模块上报：模块入口函数统一支持可选参数：
  - `progress_cb: (percent:int, status:str) -> None`
  - 只在关键阶段上报，循环场景会做“节流上报”（避免过密导致变慢）

### 6.5 使用手册可维护（外置 txt）

- **目标**：使用手册不再写死在 `ui/main_window.py`，后续可直接编辑维护。
- **实现**：
  - 文本存放在 `huancuntxt/<拼音key>.txt`
  - UI 运行时通过 `ui/components/manual_text_repo.py::load_manual_text()` 加载
- **约束**：
  - 库存处理工具（嵌入旧 UI）暂不强行统一手册机制（避免影响原逻辑）

---

### 6.6 使用统计（二级页面）

- **入口**：主界面左侧“功能导航”标题右侧按钮“使用统计”
- **页面**：`ui/components/usage_stats_page.py`（左上角返回）
- **统计规则**：
  - 每个模块有“定义节省时长”（baseline）
  - 每次成功运行记录：\( 实际节省 = max(0, baseline - runtime) \)
  - 单位统一用“小时”展示
  - 报表：按“大模块总节省时长”降序，同时展示子模块明细
- **持久化**：
  - 写入 `resources/usage_stats.json`
  - 设备迁移（指纹变化）会自动重置统计
- **埋点位置**：
  - 通用任务页：`folder_task_page.py` / `file_task_page.py`（成功回调记录）
  - 库存处理工具：在 `stock_tool_legacy_main.py` 的三项完成回调记录（见下文）
- **定义节省时长**：
  - 1.当当切出拉销量(10min)
  - 2.已推单未入库表处理(0.5h)
  - 3.发货时效表处理（0.5h）
  - 4.库存处理工具
    - A.未配货（1h）
    - B.未发货（12min）
    - C.库存报表（8min）
  - 5.退货入库时效
    - A.表的初步处理（0.5h）
    - B.表的时效计算（10min）
  - 6.盘点的分析处理(1h)
  - 7.盘点的初步处理
    - A.管易基础表处理工具1（15min）
    - B.百合基础表处理工具2（15min）
    - C.仓库实盘表处理工具3（30min）
  - 8.猴面包树B2B发货
    - A.送货单、备货单生成（每输入一个需处理的文件5min）
    - B.送货单与模板匹配（需要备货单的规格箱数）（生成的sheet数*3min）
    - C.提货单生成（记得填sdo和箱数）（生成提货单数*5min）
    - D.箱唛转换（15min）

### 7. 已接入模块清单（含输入要求与输出）

#### 6.1 当当切出拉销量（当当拉销量）

- **入口**：`modules/dangdang_sales/logic.py::process_folder`
- **输入**：选择包含待处理 `.xlsx` 的文件夹
  - 需要存在 sheet：`供应链订单明细表`
- **输出**：写入输出目录（同名/占用会自动改名）
  - 会生成处理后的文件 + `.存档.xlsx`

#### 6.2 已推单未入库表处理

- **入口**：`modules/pushed_order_not_inbound/logic.py::process_folder`
- **输入文件夹必须包含**
  - `退货商品明细汇总*.csv`
  - `店铺匹配仓库配置.xlsx`（sheet：自营仓）
- **输出**：`输出目录/已推单未入库表处理/退货未入库-截至*.xlsx`
  - 并生成附加 sheet（未审核/仓库回复）

#### 6.3 发货时效表处理

- **入口**：`modules/shipping_timeliness/logic.py::process_csv`
- **输入**：选择单个 CSV（文件名必须：`发货商品详情X月.csv`）
- **输出**：`输出目录/发货时效表处理/发货商品详情（X月）.xlsx`
  - 主表 + `24` + `48` 三个 sheet

#### 6.4 退货入库时效

- **表的初步处理**
  - **入口**：`modules/return_inbound_timeliness/step1_preprocess.py::process_step1`
  - **输入文件夹必须包含**：
    - `退货商品明细汇总*.csv`
    - `店铺匹配仓库配置.xlsx`
    - `12345.xlsx`（模板）
  - **输出**：`输出目录/退货入库时效/表的初步处理/退货入库时效分析*月总表.xlsx`
- **表的时效计算**
  - **入口**：`modules/return_inbound_timeliness/step2_calc.py::process_step2`
  - **输入**：包含 `退货入库时效分析*月总表.xlsx` 的文件夹
  - **输出**：`输出目录/退货入库时效/表的时效计算/退货商品明细汇总-*月推单.xlsx`

#### 6.5 盘点的初步处理

- **管易基础表处理工具1**
  - **入口**：`modules/inventory_preprocess/guanyi_export.py::process_folder`
  - **输入**：`商品库存导出*.csv`（可多个）+ 可选 `盘点-仓库货主匹配.xlsx`
  - **输出**：`输出目录/盘点的初步处理/管易基础表处理工具1/*.xlsx`
- **百合基础表处理工具2**
  - **入口**：`modules/inventory_preprocess/baihe_snapshot.py::process_folder`
  - **输入必须**：
    - `库存快照明细*.xlsx`（取最新）
    - `盘点-规格代码匹配.xlsx`
    - `商品库存导出*.xlsx`（用于透视表补商品名称等）
  - **输出**：`输出目录/盘点的初步处理/百合基础表处理工具2/<原文件名>-修改后.xlsx`
  - **注意**：商品名称对照依赖“工作目录（work_dir）”内的 `商品库存导出*.xlsx`
- **仓库实盘表处理工具3**
  - **入口**：`modules/inventory_preprocess/warehouse_realcount.py::process_folder`
  - **输入**：以 `信选*` / `清元*` 开头的多个 `.xlsx`（含“次品”等同前缀文件都会处理）
  - **输出**：在输出目录生成对应的 `-修改后.xlsx`（并可能更新输出目录内 `商品库存导出*.xlsx`）

#### 6.6 盘点的分析处理

- **入口**：`modules/inventory_analysis/logic.py::process_folder`
- **输入**：选择包含盘点分析所需 Excel 的文件夹
- **输出**：`输出目录/盘点的分析处理/*-修改后.xlsx`

#### 6.7 猴面包树 B2B 发货（四个子功能）

入口均为“选择输入文件夹”，输出写入统一输出目录：
- `modules/b2b_shipping/delivery_and_stock.py::process_folder`
  - 输入目录含命名形如 `易快报导单--猴面包树-...-YYYYMMDD-*.xls/.xlsx`
  - 输出：送货单/备货单（写入 output_dir）
- `modules/b2b_shipping/template_match.py::process_folder`
  - 输入目录含 `送货单模板/` 子文件夹（模板）
  - 读取 output_dir 中的 `*送货单.xlsx` 并回写
- `modules/b2b_shipping/picking_slip.py::process_folder`
  - 输入目录含 `AAA提货单模板系统专用.xlsx`
  - 读取 output_dir 中 `*送货单.xlsx` 输出 `*提货单.xlsx`
- `modules/b2b_shipping/carton_label.py::process_folder`
  - 输入目录含 `猴面包树B2B箱单.xlsx`
  - 输出：`output_dir/猴面包树箱唛-转换后.xlsx`（以“转换后”为最终产物）

#### 6.8 库存处理工具（特殊：保留原 UI/逻辑）

- **入口**：`ui/components/stock_tool_legacy_main.py`（PyQt6 适配 + 内嵌）

### 9. 发布与交付（安装包/便携版）

#### 9.1 目标
- **对外交付**：发给别人电脑后可直接安装/运行，**不依赖 Python 环境**
- **支持自定义安装路径、桌面快捷方式**
- **可选“核心代码加固/混淆”**（提高反编译成本）

#### 9.2 一站式打包目录
所有打包相关文件集中在：
- `resources/installer_tools/`

包含：
- `build_installer.bat` / `build_installer.ps1`：英文入口（避免中文路径乱码）
- `一键生成安装包.bat` / `一键生成安装包.ps1`：中文入口（内部转调英文入口）
- `installer.iss`：Inno Setup 脚本
- `requirements-build.txt`：仅打包依赖（PyInstaller/PyArmor）
- `make_ico.py`：生成 `resources/app_icon.ico`
- `清理打包中间产物.bat/.ps1`：清理 `build/`、`dist/`、`*.spec` 等中间产物

#### 9.3 产物（最终交付）
生成后输出到：
- `dist_installer/Install_ExcelTools.exe`：**安装向导**（可选安装路径 + 桌面快捷方式）
- `dist_installer/Portable_ExcelTools.zip`：**便携版**（解压即用，作为 Inno Setup 编译失败/受限时的兜底）

#### 9.4 构建命名策略（避免乱码）
- **构建名（文件/目录名）统一用英文**：`ExcelTools`
  - `dist/ExcelTools/...`
  - `build/ExcelTools/...`
  - `Install_ExcelTools.exe` / `Portable_ExcelTools.zip`
- **展示名（UI/安装界面/快捷方式）保持中文**：`Excel处理工具`

#### 9.5 “加密/加固”（密钥：072554）说明
- 本项目采用 **PyArmor 做代码加固/混淆**（提高反编译门槛，非绝对不可逆加密）
- **密钥**：`072554`
- **限制**：
  - legacy 的 `pyarmor-7 pack` **不支持 Python 3.11+**
  - 若机器存在 Python 3.10，脚本会自动创建 `.venv_build310` 用于启用加固
  - 若没有 Python 3.10，则会自动关闭加固，回退到纯 PyInstaller 打包（保证先能出安装包）

#### 9.6 清理策略（保持工程整洁）
- 打包会产生大体积中间产物：`build/`、`dist/`、`*.spec`、部分日志与缓存
- 建议每次成功出包后运行：
  - `resources/installer_tools/清理打包中间产物.bat`

### 10. 后续扩展的标准接入方式（建议流程）

- 1）从你提供的源文件提炼“核心逻辑”，迁入 `modules/<new_module>/...`
- 2）只做 I/O 适配：输入从 UI 选择、输出写入统一输出目录的独立子目录
- 3）在 `ui/main_window.py` 增加入口页面：
  - 文件夹输入用 `build_folder_task_page`
  - 单文件输入用 `build_file_task_page`
- 4）如需进度：模块入口函数支持 `progress_cb`，在关键阶段打点即可
